# FAQ - Часто задаваемые вопросы

## 🔧 Установка и настройка

### Q: Какие требования к серверу?
**A:** Минимальные требования:
- Ubuntu 22.04 (или другой Linux)
- 1 CPU core
- 512 MB RAM
- 1 GB свободного места на диске
- Интернет-соединение

### Q: Можно ли запустить на Windows/macOS?
**A:** Да, Docker работает на всех платформах. Инструкции в README рассчитаны на Ubuntu, но принцип тот же:
- Установите Docker Desktop
- Создайте `.env` файл
- Запустите `docker compose up -d`

### Q: Где получить Telegram Bot Token?
**A:** 
1. Откройте Telegram
2. Найдите @BotFather
3. Отправьте `/newbot`
4. Следуйте инструкциям
5. Скопируйте токен (формат: `123456789:ABCdef...`)

### Q: Где получить Gemini API Key?
**A:**
1. Перейдите на https://makersuite.google.com/app/apikey
2. Войдите с Google аккаунтом
3. Нажмите "Get API Key" → "Create API key"
4. Скопируйте ключ (формат: `AIzaSy...`)

### Q: Gemini API бесплатный?
**A:** Да, Google предоставляет бесплатный tier с лимитами:
- 15 запросов в минуту
- 1500 запросов в день
- Достаточно для личного использования

### Q: Нужен ли VPS или можно запустить дома?
**A:** Можно запустить где угодно:
- **VPS** - бот работает 24/7
- **Домашний ПК** - работает только когда ПК включен
- **Raspberry Pi** - отличный вариант для 24/7

## 🐛 Проблемы и решения

### Q: Бот не отвечает на сообщения
**A:** Проверьте:
```bash
# Статус контейнера
docker compose ps

# Логи
docker compose logs -f

# Перезапуск
docker compose restart
```

### Q: Ошибка "Invalid token"
**A:** Неверный Telegram Bot Token:
- Проверьте `.env` файл
- Убедитесь, что токен скопирован полностью
- Получите новый токен у @BotFather

### Q: Ошибка "API key not valid"
**A:** Неверный Gemini API Key:
- Проверьте `.env` файл
- Создайте новый ключ на Google AI Studio
- Убедитесь, что API включен в вашем проекте

### Q: Бот анализирует очень долго
**A:** Это нормально:
- Gemini API может обрабатывать 5-15 секунд
- Зависит от размера изображения
- Зависит от загрузки серверов Google

### Q: Ошибка "Rate limit exceeded"
**A:** Превышен лимит запросов:
- Бесплатный tier: 15 запросов/минуту
- Подождите минуту и попробуйте снова
- Рассмотрите платный план Google AI

### Q: Контейнер постоянно перезапускается
**A:** Проверьте логи:
```bash
docker compose logs --tail=50
```
Частые причины:
- Неверные токены в `.env`
- Отсутствует `.env` файл
- Проблемы с сетью

## 📊 Использование

### Q: Насколько точны расчеты калорий?
**A:** Приблизительная точность ±15-20%:
- AI оценивает визуально
- Зависит от качества фото
- Не учитывает точный вес
- Используйте как ориентир, а не абсолютную истину

### Q: Можно ли анализировать несколько блюд на одном фото?
**A:** Да, но лучше по отдельности:
- AI проанализирует все блюда
- Но точность будет ниже
- Рекомендуется фото каждого блюда отдельно

### Q: Какие языки поддерживает бот?
**A:** Бот отвечает на русском языке, но может анализировать:
- Этикетки на любом языке
- Анализы на любом языке
- Gemini понимает множество языков

### Q: Можно ли изменить язык ответов?
**A:** Да, отредактируйте промпты в `analyzers.py`:
```python
prompt = """You are a professional nutritionist. Analyze..."""
```

### Q: Бот запоминает историю?
**A:** Нет, каждый запрос независим:
- Нет базы данных
- Нет истории сообщений
- Каждое фото анализируется отдельно

### Q: Можно ли добавить сохранение истории?
**A:** Да, нужно доработать код:
- Добавить базу данных (SQLite/PostgreSQL)
- Сохранять результаты анализов
- Добавить команды для просмотра истории

## 🔒 Безопасность и приватность

### Q: Куда отправляются мои фото?
**A:** 
1. Telegram серверы (временно)
2. Google Gemini API (для анализа)

**A:** Риски:
- Фото проходят через Telegram и Google
- Не рекомендуется для особо конфиденциальных данных
- Используйте на свой риск

### Q: Можно ли запустить бота локально без интернета?
**A:** Нет, требуется интернет для:
- Telegram API
- Google Gemini API
- Нет офлайн-альтернативы

### Q: Кто имеет доступ к боту?
**A:**

- Можно ограничить через `ADMIN_IDS` в `.env`
- Можно добавить проверку в код

## 🛠 Техническая поддержка

### Q: Где посмотреть логи?
**A:**
```bash
# Логи Docker
docker compose logs -f

# Логи в файле
cat logs/bot.log

# Последние 100 строк
docker compose logs --tail=100
```

### Q: Как обновить бота?
**A:**
```bash
# Остановка
docker compose down

# Обновление кода (git pull или замена файлов)

# Пересборка и запуск
docker compose up -d --build
```

### Q: Как сделать бэкап?
**A:**
```bash
# Бэкап .env файла
cp .env .env.backup

# Бэкап логов
tar -czf logs_backup.tar.gz logs/

# Бэкап всего проекта
tar -czf callbot_backup.tar.gz .
```

### Q: Как перенести бота на другой сервер?
**A:**
```bash
# На старом сервере
tar -czf callbot.tar.gz callbot/

# Скопируйте на новый сервер
scp callbot.tar.gz user@new-server:/home/user/

# На новом сервере
tar -xzf callbot.tar.gz
cd callbot
docker compose up -d
```

### Q: Бот использует много памяти
**A:** Нормальное потребление:
- ~100-200 MB RAM
- Если больше - проверьте логи
- Возможна утечка памяти - перезапустите

### Q: Как мониторить бота?
**A:**
```bash

docker compose ps

# Ресурсы
docker stats nutritionist_bot

# Автоматический мониторинг
```

## 📱 Telegram специфика

### Q: Бот не видит фото
**A:** Проверьте:
- Отправляете ли вы именно фото (не файл)
- Права бота в группе (если используется в группе)
- Логи бота

### Q: Можно ли использовать бота в группах?
**A:** Да, но нужно:
- Отключить Privacy Mode у @BotFather
- Команда: `/setprivacy` → выбрать бота → Disable
- Бот будет видеть все сообщения в группе

### Q: Как узнать свой Telegram ID?
**A:**
- Напишите @userinfobot
- Или используйте @getmyid_bot
- ID нужен для `ADMIN_IDS`

### Q: Можно ли ограничить доступ к боту?
**A:** Да, в `bot.py` добавьте проверку:
```python
from config import ADMIN_IDS

async def start(update: Update, context: ContextTypes.DEFAULT_TYPE):
    if update.effective_user.id not in ADMIN_IDS:
        await update.message.reply_text("У вас нет доступа к боту")
        return
    # остальной код
```
