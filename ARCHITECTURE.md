## Поток данных

### 1. Получение фото

```
Пользователь → Telegram → Bot
                           │
                           ├─ Сохранение file_id в context
                           │
                           └─ Отправка Inline клавиатуры
                              [Калории] [Состав] [Блюдо] [Анализы]
```

### 2. Выбор типа анализа

```
Пользователь нажимает кнопку
           │
           ▼
Callback Query → Bot
                  │
                  ├─ Получение file_id из context
                  │
                  ├─ Скачивание фото через Telegram API
                  │
                  └─ Отправка "⏳ Анализирую..."
```

### 3. Анализ изображения

```
Bot → analyzers.py
       │
       ├─ Выбор метода анализа
       │  (calories/composition/food/medical)
       │
       ├─ Формирование промпта на русском
       │
       ├─ Отправка в Gemini API
       │  - Промпт (текст)
       │  - Изображение (bytes)
       │
       └─ Получение ответа от AI
```

### 4. Возврат результата

```
Gemini API → Bot → Telegram → Пользователь
                │
                └─ Логирование результата
```

## Компоненты системы

### bot.py (Главный модуль)

**Обработчики:**
- `start()` - приветствие и инструкции
- `help_command()` - подробная справка
- `handle_photo()` - получение фото и показ меню
- `button_callback()` - обработка выбора типа анализа
- `handle_text()` - обработка текстовых сообщений
- `error_handler()` - глобальная обработка ошибок

**Функции:**
- Управление состоянием через `context.user_data`
- Inline клавиатура для выбора
- Скачивание фото через Telegram API
- Разбивка длинных ответов (>4096 символов)
- Логирование всех операций

## Масштабирование

### Вертикальное (один сервер)

```
Минимальная конфигурация:
- 1 CPU core
- 512 MB RAM
- 1 GB disk
→ ~10-50 пользователей/день

Рекомендуемая конфигурация:
- 2 CPU cores
- 1 GB RAM
- 5 GB disk
→ ~100-500 пользователей/день

Мощная конфигурация:
- 4 CPU cores
- 2 GB RAM
- 10 GB disk
→ ~1000+ пользователей/день
```

### Горизонтальное (несколько серверов)

```
┌──────────────┐
│   Bot #1     │ ──┐
└──────────────┘   │
                   ├──► Telegram API
┌──────────────┐   │
│   Bot #2     │ ──┤
└──────────────┘   │
                   ├──► Gemini API
┌──────────────┐   │
│   Bot #3     │ ──┘
└──────────────┘

Каждый бот:
- Отдельный токен
- Отдельный контейнер
- Независимая работа
```
