# ✅ Чеклист для деплоя

## Перед деплоем

### 1. Подготовка API ключей

- [ ] Получен Telegram Bot Token от @BotFather
  - Откройте @BotFather в Telegram
  - Команда: `/newbot`
  - Сохраните токен (формат: `123456789:ABC...`)

- [ ] Получен Gemini API Key
  - Откройте: https://makersuite.google.com/app/apikey
  - Войдите с Google аккаунтом
  - Создайте API ключ
  - Сохраните ключ (формат: `AIzaSy...`)

- [ ] Узнан ваш Telegram ID (опционально)
  - Напишите @userinfobot
  - Сохраните ID для ADMIN_IDS

### 2. Подготовка сервера

- [ ] Есть доступ к Ubuntu 22.04 серверу
  - VPS (DigitalOcean, Linode, AWS и т.д.)
  - Или локальный компьютер с Ubuntu
  - Или Raspberry Pi с Ubuntu

- [ ] Есть SSH доступ к серверу
  ```bash
  ssh user@your-server-ip
  ```

- [ ] Сервер соответствует минимальным требованиям:
  - ✅ 512 MB RAM (рекомендуется 1 GB)
  - ✅ 1 GB свободного места (рекомендуется 5 GB)
  - ✅ 1 CPU core
  - ✅ Интернет-соединение

### 3. Подготовка файлов проекта

- [ ] Все файлы проекта скопированы на сервер
  ```bash
  # Вариант 1: через Git
  git clone <your-repo-url>
  
  # Вариант 2: через SCP
  scp -r callbot user@server:/home/user/
  
  # Вариант 3: создать архив и загрузить
  tar -czf callbot.tar.gz callbot/
  scp callbot.tar.gz user@server:/home/user/
  ssh user@server
  tar -xzf callbot.tar.gz
  ```

## Процесс деплоя

### Автоматическая установка (рекомендуется)

- [ ] Перейти в директорию проекта
  ```bash
  cd callbot
  ```

- [ ] Сделать скрипт исполняемым
  ```bash
  chmod +x install.sh
  ```

- [ ] Запустить установку
  ```bash
  ./install.sh
  ```

- [ ] Ввести API ключи когда скрипт запросит:
  - Telegram Bot Token
  - Gemini API Key
  - Admin IDs (опционально)

- [ ] Дождаться завершения установки
  - Установка Docker (~2-3 минуты)
  - Установка Docker Compose (~1 минута)
  - Сборка образа (~3-5 минут)
  - Запуск контейнера (~10 секунд)

### Ручная установка (альтернатива)

- [ ] Установить Docker
  ```bash
  curl -fsSL https://get.docker.com -o get-docker.sh
  sudo sh get-docker.sh
  sudo usermod -aG docker $USER
  newgrp docker
  ```

- [ ] Установить Docker Compose
  ```bash
  sudo apt install docker-compose-plugin -y
  ```

- [ ] Создать .env файл
  ```bash
  cp .env.example .env
  nano .env
  ```

- [ ] Заполнить .env файл:
  ```env
  TELEGRAM_BOT_TOKEN=ваш_токен
  GEMINI_API_KEY=ваш_ключ
  ADMIN_IDS=ваш_id
  ```

- [ ] Сохранить файл (Ctrl+O, Enter, Ctrl+X)

- [ ] Запустить бота
  ```bash
  docker compose up -d
  ```

## Устранение проблем

### Если бот не запускается

- [ ] Проверить логи
  ```bash
  docker compose logs
  ```

- [ ] Проверить .env файл
  ```bash
  cat .env
  ```

- [ ] Проверить токены
  - Telegram Bot Token корректный?
  - Gemini API Key корректный?

- [ ] Пересоздать контейнер
  ```bash
  docker compose down
  docker compose up -d --force-recreate
  ```

### Если бот не отвечает

- [ ] Проверить статус контейнера
  ```bash
  docker compose ps
  ```

- [ ] Перезапустить бота
  ```bash
  docker compose restart
  ```

- [ ] Проверить интернет-соединение
  ```bash
  ping -c 3 api.telegram.org
  ```

### Если анализ не работает

- [ ] Проверить Gemini API Key
  - Ключ активен?
  - Не превышен лимит запросов?

- [ ] Проверить качество фото
  - Фото четкое?
  - Размер не слишком большой?
