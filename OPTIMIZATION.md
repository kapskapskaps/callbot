# Оптимизации бота-нутрициолога

## Реализованные улучшения

### 1. Сжатие изображений
- **Библиотека**: Pillow
- **Параметры**: 
  - Максимальный размер: 800px по широкой стороне
  - Качество JPEG: 75
  - Оптимизация включена
- **Применение**: Все методы анализа фото теперь используют сжатые изображения
- **Результат**: Снижение размера передаваемых данных в API, ускорение обработки

### 2. Stateless анализ фото
- **Метод**: `analyze_photo_stateless()`
- **Особенности**:
  - Полностью без контекста переписки
  - Только фото + минимальный промпт
  - Быстрый первичный анализ
- **Промпт**: "Проанализируй еду на фото, назови блюдо и примерные КБЖУ. Будь краток."
- **Применение**: Автоматически при отправке любого фото

### 3. Команда /ask для консультаций
- **Назначение**: Серьезные профессиональные консультации
- **Ограничение контекста**: Только последние 3 сообщения
- **Стиль**: Сухой, профессиональный, без юмора
- **Лимит ответа**: 500 символов (жесткое ограничение)
- **История**: Хранится до 10 последних сообщений (5 пар вопрос-ответ)

## Использование

### Быстрый анализ фото
1. Отправьте фото
2. Получите мгновенный краткий анализ
3. При необходимости выберите детальный анализ из меню

### Профессиональная консультация
```
/ask Можно ли есть творог при повышенном холестерине?
```

## Технические детали

### Сжатие изображений (analyzers.py:17-51)
```python
def compress_image(self, image_data: bytes, max_size: int = 800, quality: int = 75) -> bytes
```
- Конвертация в RGB для JPEG
- Пропорциональное изменение размера
- LANCZOS resampling для качества
- Fallback на оригинал при ошибке

### Stateless анализ (analyzers.py:53-71)
- Без истории переписки
- Минимальный промпт
- Сжатое изображение

### Консультация с контекстом (analyzers.py:266-291)
- Ограничение: последние 3 сообщения
- Профессиональный тон
- Жесткий лимит 500 символов

## Преимущества

1. **Экономия трафика**: Сжатие изображений до 800px
2. **Скорость**: Stateless анализ работает быстрее
3. **Точность**: /ask дает профессиональные ответы без шуток
4. **Контроль затрат**: Ограничение контекста снижает потребление токенов
